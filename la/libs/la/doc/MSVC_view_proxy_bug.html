MSVC View Proxy Bug

Microsoft Visual C++ prior to and including version 9.0 erroneously issues compile errors in template view proxy operator| overloads (non-template operator| overloads are OK.) An alternative syntax is supported to work around this issue. For example, instead of m|col<C> you can use m|col<C>(). The alternative syntax is valid on all compilers, not only on MSVC.

Note however that this issue affects only template view proxies. Here is a complete list the affected view proxies: col, row, delc, delr, delrc, sw, matrix/A, vector/A.
