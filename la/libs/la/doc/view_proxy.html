Boost LA

View Proxies

Boost LA defines some functions which take a reference to a vector/matrix object on the left, a tag (in the form of function pointer) on the right, and return an object of unspecified vector/matrix type for which the vector_traits/matrix_traits template is specialized to provide a different view to the original vector/matrix object.

Usually, view proxies are invoked using operator|. For example, given a vector v, the expression v|col_matrix returns a view proxy that makes v look and feel like a matrix-column. More complex views are created by simple composition, for example v|col_matrix|transpose is equivalent to v|row_matrix.

Note: typically, the returned view proxies are simply the original object reference reinterpret_casted to an internal non-copyable vector/matrix type. This technique avoids creating a temporary object.
